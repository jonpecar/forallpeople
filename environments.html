Environments#

An environment within forallpeople is a singleton instance of the forallpeople.environment.Environment class. It is not intended to be interacted with directly but instead through the forallpeople.environment function.

Loading Environments#

Enter the name of the environment you wish to load with the environment() function.

[1]:

import forallpeople as si
si.environment('default')

Bundled environments#

forallpeople comes with the following bundled environments: * default: Defines the SI derived units * us_customary: Defines the most common units within the US Customary units system * structural: Defines a series of units in both SI and US Customary units systems that are commonly used in the field of structural engineering (the field of practice of the library author).

Environment files#

Unit environments are JSON files that describe three things: 1. The names of unit variables that are to be added to the namespace when the environment is loaded 2. The characteristics of the units that are to be recognized when the environment is loaded (i.e. dimensions, factor, symbol, and value) 3. The global default precision to display for all Physical instances (i.e. number of decimal places)

These JSON files are stored within the environments directory, a sub-directory within the forallpeople module.

When loading an environment, you are passing the file name of the JSON file without the .json extension. e.g. si.environment('default') is loading the default.json file in the environments sub-directory.

The location of your current forallpeople installation can generally be found by importing the module and then using the .__file__ attribute of the imported module’s variable name.

[4]:

import forallpeople as si
si.__file__
